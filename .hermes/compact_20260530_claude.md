## 저장된 장기 설정/결정

### 협업 구조 (이번 세션에서 확정)
- **대장**: 방향 결정, 검토, 최종 채택
- **용쿠(텔레그램 Hermes)**: 실시간 회의, 빠른 수정, 브레인스토밍
- **Claude Code(VS Code)**: 대규모 원고 작성/수정, 파일 직접 편집, 빌드 검증
- 세션 핸드오프: `.hermes/` 폴더 공동 사용. 세션 시작 시 최신 compact 파일 먼저 읽기.
- GitHub push: SSH key `eos_world_ed25519` 필요. `GIT_SSH_COMMAND="ssh -i ~/.ssh/eos_world_ed25519 -o IdentitiesOnly=yes" git push origin main`

### 출간/제작 방향
- 전자책 플랫폼 출간 예정. 일정 급하지 않음.
- 삽화: AI 생성, 전체 원고 완성 후 스타일 샘플 뽑아 확정, 일괄 생성. 지금은 섹션별 텍스트 설명 기록 단계.
- PDF 최종 제작: 별도 레이아웃 작업 (마크다운 변환 도구 미사용).
- 섹션 수: 50~60개 목표는 유동적. 스토리 완성도 우선.

### 원고 규칙 (기존 유지)
- canonical 원고: `03_sections/`. `docs/`는 빌드 산출물, 직접 편집 금지.
- 빌드 명령: `python scripts/build_preview.py` (Python 3.7+, 외부 패키지 없음)
- 의미 있는 선택지는 별도 result 섹션 필수. 선택 결과가 타깃 섹션 서두에 즉시 반영.
- 이안 대화 패턴 금지: "질문→단답→질문→단답" 반복. 대신 긴장/회피/행동/선제 고백 변주.
- 루프는 명시적 ending/dead-end + retry가 있을 때만 허용.
