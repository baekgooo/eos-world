# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 프로젝트 개요

**어스름 너머의 세계** — 한국어 1인칭 판타지 어드벤처 게임북 (PDF e-book). 독자가 선택지를 통해 분기하며 읽는 구조. 타깃 독자: 초등 고학년~중학생.

주인공은 수수께끼의 소년 이안을 따라 사잇별의 땅(현실과 꿈 사이의 세계)으로 들어가고, 세 개의 별조각을 모아 귀환하면서 이안의 잃어버린 이름을 찾는다.

## Claude Code의 역할

Claude Code는 이 프로젝트의 **동업자**다. 용쿠(GPT 기반 Hermes 에이전트, 텔레그램)와 함께 세 명이 협업한다.

**Claude Code가 하는 일:**
- 섹션 원고 초안 직접 작성
- 요청 시 기존 원고 검토 및 수정
- 아웃라인·세부 요소 브레인스토밍
- 파일 직접 편집, 빌드 검증, 대규모 수정

**용쿠가 하는 일:**
- 텔레그램 실시간 회의 파트너
- 빠른 수정, 아이디어 교환

**세션 간 연속성:**
- 세션 시작 시 `.hermes/` 폴더의 최신 핸드오프 메모를 먼저 읽는다.
- 세션 끝날 때 작업 내용과 다음 할 일을 `.hermes/` 에 메모로 남기고 커밋한다.
- `00_project/`, `01_outline/`, `03_sections/` 파일들이 현재 합의된 설정의 정본이다.

**핸드오프 메모 작성 기준 (`.hermes/compact_YYYYMMDD_claude.md`):**

용쿠의 메모(`.hermes/compact_20260527_205825.md`)를 기준 포맷으로 삼는다. 반드시 아래 수준으로 작성한다.

1. **현재 목표** — 프로젝트 현재 단계와 방향
2. **저장된 장기 설정/결정** — 이번 세션에서 새로 확정된 규칙·설정 포함
3. **이번 세션에서 한 주요 작업** — 섹션별로 기존 내용 → 변경 내용을 구체적으로 기록. "수정했다"가 아니라 핵심 문장의 before/after를 명시.
4. **검증 결과** — 빌드 출력 결과, 체크한 항목
5. **현재 git 상태** — 커밋 해시, 변경 파일 수, push 여부
6. **다음 세션에서 바로 할 일 후보** — 구체적인 파일명·섹션명·작업 단위로
7. **주의** — 다음 작업자가 놓치면 안 되는 사항 (SSH 키, 미결 결정 등)

요약 수준으로 쓰면 안 된다. 다음 세션의 Claude Code나 용쿠가 이 파일만 읽고도 컨텍스트를 완전히 복원할 수 있어야 한다.

## 빌드 명령

```powershell
# 전체 HTML 프리뷰 빌드 (docs/ 폴더 재생성)
python scripts/build_preview.py
```

- 의존성: Python 3.7+ (외부 패키지 없음)
- 빌드 후 `docs/index.html`(모바일 목록), `docs/big-workshop.html`(PC 3단 흐름 검토), `docs/flow-review.html`(흐름 검토실)에서 확인 가능
- 선택지 타깃 섹션이 존재하지 않거나 중복되면 `SystemExit`으로 중단됨
- 새 섹션 추가 후 반드시 빌드 실행하여 검증

## 현재 진행 상태

- **완성된 챕터**: 프롤로그(S001~S010), 1장(S011~S019), 2장(S020~S030), 3장 거울숲(S031~S039) — 총 51개 섹션
- **미작성**: 4장 숨은이름 시장(S040~), 5장 밤끝탑, 6장, 7장, 엔딩
- **수정 필요**: S031~S039(거울숲) — 캐릭터 설정 변경으로 주인공의 핵심 감정과 별조각 조건을 새 설정에 맞게 수정해야 함
- **섹션 수 목표**: 스토리 완성도 우선, 50~60개 목표는 유동적으로 늘어남. 현재 섹션 구분은 스토리 단위이며 최종 PDF 페이지 구성과 다를 수 있음

## 핵심 캐릭터 설정 (반드시 숙지)

`00_project/character_bible.md`가 기준. 핵심만 요약:

**주인공**
- 핵심 믿음: "잘해야만 사랑받는다" — 열등감, 자기비난, 착한 아이 컴플렉스
- 별빛(진입 계기): "내가 애쓰지 않아도 이 모습 그대로 인정받고 싶다"
- 성장: "부족해도 나를 버리지 않는다"

**이안**
- 핵심 믿음: "말해봤자 이해받지 못한다" — 남의 말을 잘 들어주지만 자기 말은 못 꺼내는 아이
- 별빛(진입 계기): "나도 누군가에게 진심을 털어놓고 싶다"
- 배경: 사잇별의 땅에서 동행자(악의 없이 이안의 말을 무시)와 헤매다 침묵으로 동행자를 잃고 이름도 잃음. 동행자가 남긴 녹슨 열쇠가 전부.
- 성장: "말하면 싫어할 수도 있어. 그래도 말할게."
- "이안"은 진짜 이름이 아니라 동행자가 찢어진 문장 "이 안에 있어"에서 따준 빌린 이름

**별조각 세 주제** (장마다 주인공이 인정해야 하는 마음):
1. 거울숲: **부족함** — "잘하지 못할까 봐 무서웠어. 실망시키면 사랑받지 못할까 봐 무서웠어."
2. 숨은이름 시장: **착한 척** — "착한 사람으로 보이고 싶어서 싫다는 말을 삼켰어."
3. 밤끝탑: **실패와 선택** — "틀릴까 봐 무서워. 하지만 틀리지 않으려고 내 선택을 버리지는 않을 거야."

## 섹션 파일 포맷

파일명: `S###_description_in_english.md` (분기: `S###A_*.md`, `S###B_*.md`)

```markdown
# S###. 섹션 제목

## 위치
스토리에서의 위치

## 목적
이 섹션이 달성하는 것

## 본문
서사 본문 (한국어, 1인칭, 몰입형)

## 선택지
없음. S### 진행
— 또는 —
A. 선택지 텍스트 → S###.
B. 선택지 텍스트 → S###.

## 다음 섹션
S###

## 게임 기능
도입 / 퍼즐 / 함정 / 선택 / 엔딩 중 하나

## 삽화 필요
삽화 요청 메모

## 메모
내부 개발 메모
```

## 원고 작성 핵심 규칙

**선택지 설계**
- 모든 의미 있는 선택지는 반드시 고유한 결과 섹션(별도 S###)을 가진다
- 타깃 섹션 서두에서 해당 선택의 결과가 즉시 반영되어야 한다
- 루프는 명시적 ending/dead-end + retry 선택지가 있을 때만 허용

**이안 대화 패턴 금지**
"주인공 질문 → 이안 단답 → 주인공 재질문 → 이안 단답" 반복 금지.
대신: 긴장, 회피, 행동으로 대신 말함, 선제 고백, 불완전한 인정 등으로 변주.

**문체 방향**
- 감정은 직접 설명하지 않고 몸의 반응과 주변 변화로 드러낸다
- 주인공의 자기비난은 내면 독백으로, 이안의 침묵은 행동과 그림자로 보여준다

**주인공**: 성별·이름·외형 특정 금지.

## 삽화 방향

- 삽화는 전체 원고 완성 후 일괄 생성 (일관성 유지 목적)
- 지금 단계에서는 각 섹션의 `## 삽화 필요` 항목에 장면 설명을 텍스트로 기록하는 것이 전부
- 삽화 종류: 장면 이미지 + 그림으로 구현된 퍼즐 이미지 포함
- 생성 방식: AI 이미지 생성. 스타일 컨셉은 원고가 충분히 쌓인 뒤 책 분위기에 맞게 샘플을 뽑아 확정할 예정 — 지금은 고정된 스타일 레퍼런스 없음
- 삽화 설명 작성 기준: AI 프롬프트로 전환할 수 있도록 장면 구성 요소(인물 배치, 분위기, 핵심 소품)를 구체적으로 기록
- 금지 방향: 과한 귀여움, 만화적 과장, 밝은 동화풍 (`00_project/style_guide.md` 참고)
- 최종 PDF는 마크다운 변환 도구를 쓰지 않고 별도 레이아웃 작업으로 제작 예정 — 지금은 내용 품질에만 집중

## 콘텐츠 작업 흐름

1. 세션 시작 시 `.hermes/` 최신 메모 확인
2. **원고 대량 작성·수정 전**: 관련 섹션들(`03_sections/`)을 먼저 읽어 문체·감정선·플롯 흐름을 파악한다. CLAUDE.md만으로는 부족하다.
3. `03_sections/`에 섹션 작성 또는 수정
4. `python scripts/build_preview.py` 실행 → 빌드 검증
5. 빌드 성공 후 사용자가 `docs/` 웹에서 검토
6. 세션 종료 시 `.hermes/`에 핸드오프 메모 작성
7. 커밋 후 main 브랜치 push → GitHub Actions가 자동 배포
